# Contributing

Thanks for contributing to `meta-autoresearch`.

This repository is an early-stage research space. The main goal is not fast consensus. It is disciplined exploration. Contributions should help the inquiry become clearer, more rigorous, and more useful.

## Principles

- preserve the difference between framing, hypothesis, and conclusion
- make assumptions explicit
- prefer traceable reasoning over rhetorical force
- record uncertainty instead of smoothing it away
- treat AI output as material for judgment, not as judgment itself

## Writing guidelines

- write clearly and directly
- keep philosophical language grounded in operational consequences
- avoid overstating certainty
- avoid treating novelty as proof
- keep the tone serious but not pompous

## Source handling

- cite sources when summarizing evidence or making empirical claims
- distinguish between direct evidence, interpretation, and speculation
- note when a claim depends on limited or uncertain evidence
- do not launder unsupported claims through confident prose

## Editing workflow

- prefer small, legible changes
- explain why a change improves the method, framing, or research usefulness
- preserve the project's distinction between public overview docs and working research materials
- if adding a new concept, define it in `docs/glossary.md` or use existing terms consistently
- use date-first, lowercase, kebab-case filenames for research artifacts

## Working areas

- see `research/README.md` for workspace structure and naming conventions
- use `research/notes/` for exploratory notes and source synthesis
- use `research/scenarios/` for scenario drafts and recombinations
- use `research/experiments/` for process tests and evaluation trials

## What to avoid

- presenting speculative outputs as settled conclusions
- narrowing the inquiry too early around a preferred narrative
- adding unnecessary product assumptions before the method is ready
- confusing eloquence with rigor

## Pull request guidance

If you open a pull request, describe:

- what changed
- why it helps
- whether it affects framing, method, or active research
- any open uncertainties introduced by the change
