# Loop Runs

Use this directory for auditable records of one full research cycle.

Start from `research/loops/TEMPLATE.md` when documenting a loop run.

Use date-first, lowercase, kebab-case filenames such as `2026-03-27-whiplash-loop-run.md`.

Good candidates for this folder:

- one complete branch cycle from framing through evaluation
- a comparative record of how a loop changed after revision
- a maturity assessment before repeating or scaling a loop

Loop runs are not just summaries. They should make the full cycle legible: what entered the loop, what changed, what survived, what was pruned, and what the method learned.
