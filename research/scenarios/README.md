# Scenarios

Use this directory for structured scenario drafts, variants, and recombinations.

Start from `research/scenarios/TEMPLATE.md` when creating a new scenario.

Use date-first, lowercase, kebab-case filenames such as `2026-03-27-compound-seasonal-whiplash.md`.

Each scenario should aim to capture:

- the question or domain slice being explored
- the assumptions in play
- the logic of the scenario
- the relevant signals or evidence classes
- the reason it is worth keeping or testing further

Scenarios are exploratory artifacts, not forecasts.
