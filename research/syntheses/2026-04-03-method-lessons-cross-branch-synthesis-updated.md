## Structure Type Stability

### What We Have

Three stable structure types, each validated across multiple domains:

| Structure Type | Climate Anchor | Non-Climate Validation | Key Insight |
|----------------|---------------|------------------------|-------------|
| **Sequence failure** | whiplash | HPAI (biological transmission chain) | Risk becomes visible only when conditions are understood as a chain, not separate events |
| **Correlation/transmission** | breadbasket | HPAI (trade restriction amplification), wealth-concentration (financial concentration) | Risk becomes visible only when distributed nodes are understood as one coupled system |
| **Design/rule conflict** | hydrologic | HPAI (biosecurity inadequacy), wealth-concentration (asset regime mismatch) | Risk becomes visible when inherited rules no longer fit changed system dynamics |

### What We Learned

1. **Structure types emerge from grounding, not framing**
   - All five branches started as vague scenario families
   - Named cases made the structures legible
   - Structure types are discovered through iteration, not imposed at the start

2. **Structure types are not mutually exclusive**
   - Hydrologic initially looked like sequence failure
   - Wealth-concentration and avian-flu-zoonotic arrived as hybrids
   - The same domain can surface multiple structures at different levels of analysis

3. **Structure types need bounded cases, not broad scenarios**
   - Parent scenarios are too broad to be method-shaping
   - Bounded variants carry the structural signal
   - Method-shaping happens at the variant level, not the parent level

4. **Hybrid structures can have multiple component counts**
   - Wealth-concentration: 2 components (transmission + rule-conflict)
   - Avian-flu-zoonotic: 3 components (correlation + sequence + design/rule)
   - Component count correlates with grounding complexity

## Non-Climate Portability: Updated

### Two Non-Climite Branches

| Branch | Domain | Domains Tested | Structure | Maturity |
|--------|--------|---------------|-----------|----------|
| **wealth-concentration** | Political economy | Finance, compute | 2-component hybrid | L4 |
| **avian-flu-zoonotic** | Biological volatility | HPAI zoonotic transmission | 3-component hybrid | L4 |

### Portability Arrives in Stages (Updated)

| Stage | Description | Wealth-Concentration | Avian Flu |
|-------|-------------|---------------------|-----------|
| 1. Thematic similarity | "This also involves risk" | ✅ | ✅ |
| 2. Structural resonance | "This also has transmission dynamics" | ✅ | ✅ |
| 3. Hybrid mapping | "This overlaps multiple structures" | ✅ | ✅ |
| 4. Structure resolution | "This is primarily X, secondarily Y" | ⏳ | ⏳ |
| 5. Method-shaping | "This changes how we understand structure X" | ✅ | ✅ |

**Both branches reached method-shaping without structure resolution** — hybrid is stable, not intermediate.

### Key Findings

1. **Two non-climate domains confirm portability** — Wealth-concentration (political economy) and avian-flu-zoonotic (biological volatility) both demonstrate the method travels.

2. **Hybrid is the natural non-climate form** — Both non-climate branches arrived as hybrids. Clean structure mapping may be a climate-specific phenomenon.

3. **Biological volatility introduces new dynamics** — Viral evolution, environmental reservoir, host jumping, intervention points, faster timescale. The method accommodates these but structure types need biological adaptation.

4. **Multi-component hybrids are possible** — HPAI's 3-component hybrid is more complex than wealth-concentration's 2-component hybrid. Component count matters for grounding requirements.

### Strengths

1. **Resists premature closure**
   - Five branches, no single answers
   - Target fixation reduced by design

2. **Surfaces hidden structures**
   - Three structure types discovered through grounding
   - Subtype taxonomies refine each structure

3. **Makes curation explicit**
   - 9 discard records across branches
   - Keep/revise/discard decisions documented

4. **Supports cross-domain transfer**
   - Two non-climate domains validated
   - Structure types are domain-agnostic

5. **Preserves audit trail**
   - 16 loop-run records
   - Branch manifests track state over time

6. **Accommodates biological volatility**
   - Viral evolution, environmental reservoir, host jumping
   - Multi-component hybrids validated

## Where the Method Struggles

### Weaknesses and Open Questions

1. **Measurement remains provisional**
   - Qualitative + quantitative scoring
   - Evaluation criteria can drift

2. **Scenario proliferation without selection**
   - Breadbasket has 7 variants
   - Soft curation accumulates overhead

3. **Hybrid resolution is slow**
   - Both non-climate branches remain hybrid
   - No "resolve hybrid" pass type exists

4. **Method influence is hard to achieve**
   - 4 of 5 branches have achieved method-shaping
   - Avian-flu-zoonotic just reached L4

5. **Token costs limit cycle frequency**
   - ~$0.003/cycle, ~25s per cycle
   - Sustainable but not free

## What to Stop, Keep, and Start

### Stop Doing

1. **Assuming structure types are mutually exclusive**
2. **Treating parent scenarios as method-shaping**
3. **Expecting clean non-climate mapping on first pass**
4. **Accumulating without pruning**
5. **Treating all hybrids as equivalent** — 2-component differs from 3-component

### Keep Doing

1. **Grounding in named cases**
2. **Explicit curation with keep/revise/discard**
3. **Loop-run records**
4. **Source discipline**
5. **Human/AI division of labor**
6. **Cross-branch comparison every 2-3 branches**

### Start Doing

1. **Track component count for hybrids** — 2-component vs. 3-component vs. more
2. **Document subtype taxonomies** — Each branch has discovered subtypes; these should be formalized
3. **Accommodate biological volatility** — Viral evolution, environmental reservoir, intervention points, faster timescale
4. **Model performance monitoring** — Monthly benchmark, log switches
5. **Define L5 criteria** — What beyond method-shaping?
