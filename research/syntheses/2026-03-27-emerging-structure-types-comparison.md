# Emerging Structure Types Comparison

- title: Emerging Structure Types Comparison
- date: 2026-03-27

- type: method synthesis

## Purpose

Compare the three emerging structure types now visible in the repository so the project can distinguish what each one is, what kind of evidence it needs, and what a branch must show before the method should treat it as a meaningful structural discovery.

## The three emerging types

### 1. Sequence failure

Core idea: the risk becomes visible only when conditions are understood as a chain rather than as separate events, phases, or hazard categories.

Current lead branch:

- `whiplash`

### 2. Correlation/transmission failure

Core idea: the risk becomes visible only when distributed nodes are understood as a coupled system rather than as separate local problems.

Current lead branch:

- `breadbasket`

### 3. Design/rule conflict under volatility

Core idea: the risk becomes visible when inherited rules, thresholds, allocation frameworks, or infrastructure purposes no longer fit the system dynamics they are expected to govern.

Current lead branch:

- `hydrologic`

## Core contrast

### Sequence failure asks

`What changes when time is misread?`

Its central failure is interpretive transition.

### Correlation/transmission failure asks

`What changes when system coupling is misread?`

Its central failure is distributed interdependence.

### Design/rule conflict under volatility asks

`What changes when the formal system design no longer fits the system's actual behavior?`

Its central failure is mismatch between inherited governing logic and changed conditions.

## What each type tends to uncover

### Sequence failure

- category breakdown
- transition blindness
- overconfidence in relief narratives
- institutional lag across phases

### Correlation/transmission failure

- false confidence in substitutability
- hidden dependence on buffers and fallback nodes
- uneven downstream exposure
- amplification through policy, trade, stocks, or procurement

### Design/rule conflict under volatility

- hidden dependence on legacy thresholds and operating assumptions
- conflict between multiple formal purposes inside one system
- repeated emergency adaptation as a sign of deeper structural mismatch
- brittleness in rule architecture rather than only in physical assets

## What kind of evidence each type needs most

### Sequence failure

- chronology
- evidence of transition misreading
- named cases where one phase changes the meaning of the next

### Correlation/transmission failure

- node-role clarity
- evidence of coupled stress or reduced substitutability
- transmission evidence from stress to downstream consequence

### Design/rule conflict under volatility

- explicit rule or threshold architecture
- evidence that operating assumptions are under strain
- documentation of redesign, exceptions, contingency layers, or governance stress under changed conditions

## What counts as enough to treat a type as real

### Minimum signs

- more than one branch artifact points toward the same hidden structure
- the structure improves explanation, not just naming
- the structure changes what the method looks for next
- the structure supports different judgments than a purely topical framing would

By that standard:

- `sequence failure` is real enough to use
- `correlation/transmission failure` is real enough to use
- `design/rule conflict under volatility` is now strong enough to use as an emerging structure type, though it should still be revised if later branches show the distinction is weaker than it currently appears

## Current branch alignment

### Whiplash

- strongest structure: sequence failure
- current need: tighter chronology and more explicit institutional interpretation evidence

### Breadbasket

- strongest structure: correlation/transmission failure
- current need: deeper importer and reserve comparison at the downstream layer

### Hydrologic

- strongest current candidate: design/rule conflict under volatility
- current need: deepen the branch without collapsing it back into a generic subtype, so the new structure remains pressure-tested rather than merely declared

## Method implication

The project is now doing something more specific than scenario generation.

It is building a catalog of structural ways inquiry fails when systems move faster than institutional framing does. That is a stronger methodological claim than "we can write good scenarios."

## Caution

The danger now is premature ontology.

Just because a structure type is useful does not mean it is final. The right posture is:

- name it provisionally
- test it against more than one branch or case
- revise or collapse it if later evidence shows the distinction is weak

## Next move

1. add a branch-maturity rubric so the three current branches can be judged more consistently
2. use that rubric to assess whether hydrologic is mature enough to support a distinct structure label
3. only then consider whether a fourth structure is actually emerging or whether the current three just need deeper testing
