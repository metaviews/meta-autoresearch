## Structure Type Stability

### What We Have

Three stable structure types, each anchored by a climate branch:

| Structure Type | Anchor Branch | Key Insight |
|----------------|---------------|-------------|
| **Sequence failure** | whiplash | Risk becomes visible only when conditions are understood as a chain, not separate events |
| **Correlation/transmission** | breadbasket | Risk becomes visible only when distributed nodes are understood as one coupled system |
| **Design/rule conflict** | hydrologic | Risk becomes visible when inherited rules no longer fit changed system dynamics |

### What We Learned

1. **Structure types emerge from grounding, not framing**
   - All three started as vague scenario families
   - Named cases (Feather River, Russia-Europe wheat, SEQ water grid) made the structures legible
   - Structure types are discovered through iteration, not imposed at the start

2. **Structure types are not mutually exclusive**
   - Hydrologic initially looked like sequence failure (wet-to-dry transition)
   - Grounding revealed the deeper issue: infrastructure rules built for stationarity facing non-stationarity
   - The same domain can surface multiple structures at different levels of analysis

3. **Structure types need bounded cases, not broad scenarios**
   - Parent scenarios (e.g., `compound-seasonal-whiplash.md`) are too broad to be method-shaping
   - Bounded variants (e.g., `upper-colorado-recovery-to-shortage-whiplash.md`) carry the structural signal
   - Method-shaping happens at the variant level, not the parent level

## Non-Climate Portability: What We Learned

### The Wealth-Concentration Result

The first non-climate branch reached **Level 3 (comparative)** with these characteristics:

- ✅ Multiple grounded variants (US asset-regime, India credit squeeze)
- ✅ Internal comparison structure (evaluation matrix, lead comparison)
- ✅ Explicit curation decisions (keep/revise judgments)
- ✅ Loop-run record with audit trail
- ✅ Discard records with reasoning

**But not yet Level 4 because:**
- ❌ Hybrid structure (transmission + rule-conflict) rather than clean structure signal
- ❌ Method influence is limited to "hybrids are possible" rather than changing how structures work
- ❌ No clear reusable pattern for future non-climate branches

### The Hybrid Finding

**Initial expectation:** Non-climate branches would map cleanly to one existing structure type.

**Actual result:** The first non-climate branch arrived as a hybrid.

**Method implication:** This is not a failure—it's a discovery about how portability works:

1. **Portability arrives in stages:**
   - Stage 1: Thematic similarity ("this also involves risk")
   - Stage 2: Structural resonance ("this also has transmission dynamics")
   - Stage 3: Hybrid mapping ("this overlaps multiple structures")
   - Stage 4: Structure resolution ("this is primarily X, secondarily Y")
   - Stage 5: Method-shaping ("this changes how we understand structure X")

2. **Hybrids are legitimate intermediate forms:**
   - Wealth-concentration is not "failed" because it's hybrid
   - It's doing the work that non-climate branches need to do: testing whether structures travel
   - The hybrid finding itself is methodologically valuable

3. **Climate branches also started hybrid:**
   - Hydrologic initially looked like sequence failure
   - Only grounding revealed the design/rule conflict as the deeper structure
   - Wealth-concentration may resolve similarly with more grounding

### What Portability Requires

Based on wealth-concentration, non-climate portability needs:

1. **At least 2-3 grounded variants** with named cases
2. **Internal comparison** that distinguishes strongest/weakest
3. **Explicit curation** with keep/revise/discard decisions
4. **Structure mapping** that tests against existing types
5. **Method-level reflection** on what the branch teaches about structure

Wealth-concentration has all five. The question is whether hybrid status disqualifies it from L4.

## Level 3 vs. Level 4: Explicit Criteria

### Level 3 (Comparative) - wealth-concentration has achieved this

**Structural requirements:**
- [x] 2+ meaningful variants (wealth-concentration has 4)
- [x] Grounded in named cases (US asset-regime, India credit)
- [x] Internal comparison exists (5 syntheses)
- [x] Strongest/weakest variants identified
- [x] Loop-run record exists
- [x] Discard records exist

**Curation requirements:**
- [x] At least one explicit keep/revise/discard judgment
- [x] Evaluation matrix applied to all variants
- [x] Clear next step identified

### Level 4 (Method-Shaping) - the bar wealth-concentration hasn't cleared

**Structural requirements:**
- [x] All Level 3 criteria met
- [ ] Supports a reusable structure type (or clarifies how hybrids work)
- [ ] Loop-run record is strong enough to be reused as a template
- [ ] Branch influences method documents, not just research artifacts

**Method influence requirements:**
- [ ] The branch changes how we understand at least one structure type
- [ ] The branch produces a generalizable lesson for future branches
- [ ] The branch's structure mapping is stable enough to guide future non-climate work

### The Gap

Wealth-concentration's gap is not structural—it has all the artifacts. The gap is **method influence**:

1. **Structure type question unresolved:**
   - Is "hybrid" itself a reusable structure type?
   - Or is hybrid an intermediate state that resolves with more grounding?
   - The method doesn't yet know, and wealth-concentration alone can't answer this

2. **Template reusability unclear:**
   - Could another researcher use wealth-concentration's loop-run as a template?
   - The loop record exists, but hasn't been tested for reusability

3. **Method document influence:**
   - This synthesis is the first method document that wealth-concentration has shaped
   - Prior to this, the three structure types were stable without wealth-concentration's input

## Recommendation: Wealth-Concentration at Level 4 (with caveats)

**Judgment:** Promote wealth-concentration to **Level 4**, but with an explicit caveat about hybrid status.

**Rationale:**

1. **The hybrid finding IS method-shaping:**
   - Before wealth-concentration, the method assumed clean structure mapping
   - The hybrid discovery changes how future non-climate branches will be evaluated
   - This is genuine method influence, not just domain content

2. **The branch has all structural prerequisites:**
   - Grounded variants, comparison, curation, loop record, discards
   - The evaluation matrix explicitly produced a `revise` judgment (private-credit scenario)
   - This is the kind of curation pressure that defines L4

3. **The caveat is the contribution:**
   - Level 4 status comes with an explicit note: "hybrid portability, not resolved"
   - Future branches can now test: do they also start hybrid, or is this unique to wealth-concentration?
   - The caveat itself becomes a method tool

**Updated maturity note:**
```
Level 4: method-shaping (hybrid portability demonstrated)
```

**What this changes:**
- The method now expects non-climate branches may start as hybrids
- Structure mapping should test for hybrid status, not assume clean mapping
- Future L3→L4 promotions can use wealth-concentration as the hybrid precedent

## What the Method Should Stop, Keep, and Start

### Stop Doing

1. **Assuming structure types are mutually exclusive**
   - Hydrologic and wealth-concentration both show overlap is possible
   - Stop forcing branches into single-structure boxes

2. **Treating parent scenarios as method-shaping**
   - The structural signal lives in bounded variants, not parent scenarios
   - Stop evaluating branches by their parent scenario quality

3. **Expecting clean non-climate mapping on first pass**
   - Wealth-concentration shows hybrids are legitimate intermediate forms
   - Stop treating hybrid status as failure

### Keep Doing

1. **Grounding in named cases**
   - All four branches show this is the path from vague to legible
   - Keep requiring at least 2-3 grounded variants per branch

2. **Explicit curation with keep/revise/discard**
   - The evaluation matrices produce real decisions, not just scores
   - Keep requiring at least one `revise` or `discard` per mature branch

3. **Loop-run records**
   - All four branches have auditable histories
   - Keep requiring loop documentation for L3+

### Start Doing

1. **Expect and document hybrid structures**
   - Add "hybrid" as a legitimate structure_type value
   - Start requiring structure mapping to test for overlap, not just single-type fit

2. **Cross-branch synthesis every 2-3 branches**
   - This synthesis is the first of its kind
   - Start requiring cross-branch reflection as branches accumulate

3. **Method influence as explicit L4 criterion**
   - Add "influences method documents" to L4 criteria in branch-maturity.md
   - Start tracking which branches change the method vs. populate it

## Open Questions

1. **Will wealth-concentration's hybrid status resolve with more grounding?**
   - If yes, hybrid is intermediate
   - If no, hybrid is a stable non-climate form

2. **How many non-climate branches are needed to generalize?**
   - One branch (wealth-concentration) shows portability is possible
   - Two branches would show it's reproducible
   - Three branches would show it's a pattern

3. **What domain should be next for non-climate proving-ground?**
   - Political economy (wealth-concentration) tests one domain
   - A third domain (health? technology? finance?) would test method generality

4. **When does a branch stop expanding and redirect?**
   - Breadbasket has 5 variants—is there such a thing as too many?
   - Should L4 branches have a "maturity ceiling" that triggers redirection?
